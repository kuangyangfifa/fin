# 💼 财务管理系统前端优化设计说明

## 1. 首页 Dashboard 数据可视化升级
**目标**：将枯燥的财务数字转化为直观的决策图表，实现“一眼看清财务状况”。

### 1.1 引入图表库
- **技术选型**：Chart.js (轻量级、性能好、支持响应式)。
- **引入方式**：CDN 引入 `https://cdn.jsdelivr.net/npm/chart.js`。

### 1.2 核心看板指标
1.  **📊 月度收支趋势图 (Line/Bar Combined Chart)**
    - **位置**：Dashboard 第一行统计卡片下方，占据 8/12 宽度。
    - **指标**：展示过去 6 个月的“总收入”与“总支出”柱状对比，并叠加一条“净利润”折线。
    - **价值**：直观反映公司现金流的增长与收缩趋势。
2.  **🍩 账户资产占比图 (Doughnut Chart)**
    - **位置**：Dashboard 第一行统计卡片下方，占据 4/12 宽度。
    - **指标**：各账户（如：公司基本户、招行、支付宝等）当前余额占比。
    - **价值**：快速识别资金分布，评估资金集中风险。

## 2. 流水查询表格响应式重构
**目标**：解决“列数过多导致移动端体验极差”的问题。

### 2.1 桌面端布局优化 (PC)
- **紧凑模式**：调小表格基础字号 (0.875rem)，缩短 Cell 内边距。
- **按需展示**：对非核心备注（备注 3、4、5）使用 Bootstrap 响应式类 `d-none d-xxl-table-cell`，仅在超宽屏显示。
- **固定表头**：实现 `sticky-top` 效果，长列表滚动时表头不动。

### 2.2 移动端卡片流 (Mobile < 768px)
- **交互切换**：隐藏原生 `<table>`，改为 `flex-column` 布局的卡片流。
- **单条记录展示样式**：
    ```text
    +------------------------------------------+
    | 2024-03-27                      [编辑][删除] |
    | 摘要内容摘要内容...                           |
    | [账户名]                          [+¥1,000] |
    | [标签1] [标签2]                     余额: ¥xxx |
    +------------------------------------------+
    ```
- **FAB 按钮**：右下角增加悬浮“+”按钮，方便手机单手操作录入。

## 3. 录入表单交互增强 (note1/note2 联动)
**目标**：降低用户思考维度，减少录入错误。

### 3.1 note1 与 note2 智能联动
- **逻辑定义**：
    - `note1`（大类）决定 `note2`（子类）的可选范围。
    - 若用户未选择 `note1`，`note2` 处于禁用状态或显示提示。
    - **实时校验**：当收入/支出均为 0 时，提交按钮保持置灰，并以 Tooltip 形式提示用户。
- **互斥动画**：当用户在“收入”框输入非零数字时，“支出”框自动清零并伴随一个“淡出/禁用”的视觉反馈。

---

## 4. 视觉规范与 UI 动效
### 4.1 登录页沉浸式升级
- **设计稿建议**：背景采用 `#1e1b4b` 到 `#312e81` 的深色渐变，中间登录框采用 `backdrop-filter: blur(10px)` 实现毛玻璃效果。

### 4.2 微动效 (Micro-interactions)
- **悬浮反馈**：所有 `card` 和 `btn` 在悬浮时增加 `transform: translateY(-4px)` 和 `box-shadow` 加深。
- **加载态**：图表加载时显示 Spinner 占位符。
- **侧边栏**：移动端唤起时使用流畅的 Slide 动画。

---

## 5. 验收标准
1.  **PC端**：首页图表能正确渲染且支持点击交互。
2.  **移动端**：iPhone 13 宽度下流水列表无横向滚动条，以卡片形式完整展示核心信息。
3.  **录入页**：选择不同的 `note1` 后，`note2` 的下拉候选项实时更新，且无逻辑延迟。
